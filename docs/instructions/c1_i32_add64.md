## `i32_add64`

### **Description**

The `i32_add64` instruction **adds two 32-bit integers and produces a 64-bit result**. It pops two 32-bit integers from the stack, extends them to 64-bit signed integers, performs addition, and pushes the 64-bit result back onto the stack.

### **Behavior**

1. **Pops** the second operand (right-hand side) from the stack.
2. **Pops** the first operand (left-hand side) from the stack.
3. **Extends** both values to 64-bit signed integers.
4. **Performs** 64-bit addition.
5. **Pushes** the 64-bit result onto the stack.
6. **Increments** the instruction pointer (`ip`) by 1.

### **Registers and Memory Changes**

- **Instruction Pointer (`ip`)**: Increased by 1.
- **Stack Pointer (`sp`)**: **Unchanged** (two 32-bit values are popped, one 64-bit value is pushed).
- **Memory**: **Unchanged**.

### **Stack Changes**

#### **Before Execution:**

```
[ ... | lhs | rhs | sp ]
```

#### **After Execution:**

```
[ ... | result_64 | sp ]
```

Where `result_64` is `(int64_t)(int32_t)lhs + (int64_t)(int32_t)rhs`.

### **Operands**

- **None** (This instruction does not take any operands).

### **Notes**

- This is a **binary operation** that consumes two 32-bit stack values and produces one 64-bit result.
- **No traps** can occur during execution of this instruction.
- The operands are **sign-extended** to 64 bits before addition.
- The result is a **full 64-bit sum**, so no overflow occurs.
- Equivalent to the expression `(int64_t)(int32_t)lhs + (int64_t)(int32_t)rhs` in high-level languages.
- The operation is **commutative**: `a + b = b + a`.
- Useful for high-precision arithmetic where overflow detection/prevention is needed.
