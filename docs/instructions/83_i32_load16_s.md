## `i32_load16_s`

### **Description**

The `i32_load16_s` instruction **loads a signed 16-bit integer from memory and sign-extends it to 32 bits**. It pops a memory address from the stack, adds the offset, loads a signed 16-bit integer from the computed address, extends it to 32 bits with sign extension, and pushes the result onto the stack.

### **Behavior**

1. **Pops** the memory address from the stack.
2. **Computes** the effective address by adding the offset to the popped address.
3. **Checks** if the effective address is within memory bounds.
4. **Loads** a signed 16-bit integer from memory at the computed address.
5. **Sign-extends** the loaded value to 32 bits.
6. **Pushes** the extended value onto the stack.
7. **Increments** the instruction pointer (`ip`) by 1.
8. If the memory access is **out of bounds**, execution **traps** with `TrapCode::MemoryOutOfBounds`.

### **Registers and Memory Changes**

- **Instruction Pointer (`ip`)**: Increased by 1 (or execution halts on trap).
- **Stack Pointer (`sp`)**: **Unchanged** (one value is popped, one is pushed).
- **Memory**: **Read** at the computed memory address.

### **Stack Changes**

#### **Before Execution:**

```
[ ... | address | sp ]
```

#### **After Execution (Successful Load):**

```
[ ... | loaded_value | sp ]
```

#### **After Execution (Memory Out of Bounds - Trap):**

- **Execution halts** due to a `MemoryOutOfBounds` trap.

### **Operands**

- `offset` (AddressOffset): A constant offset added to the memory address before loading.

### **Notes**

- **Can trap** if the effective address is beyond the allocated memory range.
- The offset is applied **before** memory access and should be within valid bounds.
- **Does not modify memory**, only reads from it.
- Loads exactly 2 bytes (16 bits) from memory.
- The loaded value is interpreted as a **signed 16-bit integer** and sign-extended to 32 bits.
- **Sign extension** means if bit 15 is 1, the upper 16 bits are filled with 1s; if bit 15 is 0, they are filled with 0s.
- Result range: [-32768, 32767].
- Memory is accessed in **little-endian** byte order.
- Equivalent to loading a `int16_t` from memory at `address + offset` and casting to `int32_t`.
